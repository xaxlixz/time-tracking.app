План спринтов
Спринт 1 (День 1): Установка и настройка окружения
Цель: Подготовить все инструменты для разработки и создания проекта.

Установка Python:

Установи Python 3.8+ с официального сайта python.org.
Убедись, что при установке поставил галочку Add Python to PATH.
Установка Visual Studio Code (VS Code):

Скачай и установи VS Code с официального сайта.
Установи расширения: Python и Django.
Установка PostgreSQL:

Установи PostgreSQL с официального сайта.
Убедись, что настроил базу данных через pgAdmin и создал базу timetrackapp_db.
Установка Git и создание репозитория на GitHub:

Установи Git с официального сайта.
Зарегистрируйся на GitHub и создай новый репозиторий.
Создание проекта:

Создай папку проекта и настрой виртуальное окружение в VS Code через терминал:
bash
Копировать код
mkdir time-tracking-app
cd time-tracking-app
python -m venv venv
venv\Scripts\activate
Спринт 2 (День 2): Создание базового проекта и системы контроля версий
Цель: Инициализировать проект и подключить его к системе контроля версий.

Инициализация проекта Django (или Flask):

Установи Django:
bash
Копировать код
pip install django
Создай новый проект:
bash
Копировать код
django-admin startproject config .
Настройка базы данных PostgreSQL:

В файле settings.py проекта добавь настройки базы данных:
python
Копировать код
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': 'timetrackapp_db',
        'USER': 'postgres',
        'PASSWORD': 'your_password',
        'HOST': 'localhost',
        'PORT': '5432',
    }
}
Применение миграций:

Выполни команды:
bash
Копировать код
python manage.py migrate
Настройка Git и GitHub:

Выполни инициализацию Git в папке проекта:
bash
Копировать код
git init
Добавь проект в удаленный репозиторий GitHub.
Спринт 3 (День 3): Регистрация и авторизация пользователей
Цель: Реализовать базовую систему регистрации и авторизации.

Создание модели пользователя:

Если используешь Django, настрой стандартную модель пользователей.
Создание формы регистрации:

Создай HTML-форму с полями: электронная почта, пароль, телефон, роль (рабочий, директор, арендатор).
Добавление логики регистрации:

Обработай форму регистрации и сохрани данные пользователя в базу данных.
Создание формы авторизации:

Реализуй форму для входа пользователя.
Спринт 4 (День 4): Подтверждение электронной почты и восстановление пароля
Цель: Реализовать проверку электронной почты и восстановление пароля.

Настройка отправки писем:

Настрой сервер для отправки писем через SMTP для подтверждения аккаунта и восстановления пароля.
Создание логики подтверждения:

Реализуй функционал для подтверждения почты после регистрации.
Восстановление пароля:

Создай форму для восстановления пароля и логику отправки временного пароля на почту.
Спринт 5 (День 5): Реализация ролей пользователей и их интерфейсов
Цель: Реализовать разделение интерфейсов по ролям (рабочий, директор, арендатор).

Создание ролей пользователей:

Раздели права доступа к интерфейсам на основе роли пользователя.
Создание разных интерфейсов:

Для каждой роли добавь отдельный интерфейс с нужными возможностями (ввод времени, управление пользователями и т.д.).
Спринт 6 (День 6): Ввод времени работы и его валидация
Цель: Создать интерфейс для ввода рабочего времени.

Создание формы для ввода времени:

Рабочие могут вводить своё рабочее время, дату начала и окончания.
Валидация введенных данных:

Добавь проверку корректности времени работы.
Спринт 7 (День 7): Запрос подтверждения времени и уведомления
Цель: Реализовать процесс отправки запросов на подтверждение времени.

Рабочие могут отправить запрос на подтверждение времени.
Арендатор и директор получают уведомления и могут подтвердить или отклонить запрос.
Спринт 8 (День 8): История переписки и комментарии
Цель: Добавить возможность оставлять комментарии к запросам.

Рабочие и арендаторы могут добавлять комментарии.
Система ведёт историю переписки.
Спринт 9 (День 9): Уведомления и эскалация непросмотренных запросов
Цель: Автоматизировать уведомления и эскалацию для непросмотренных запросов.

Спринт 10 (День 10): Экспорт данных и отчетность
Цель: Реализовать экспорт данных в Excel и генерацию отчетов.

Спринт 11 (День 11): Тестирование, отладка и финальная подготовка к запуску
Цель: Полное тестирование системы, исправление ошибок и финальная подготовка к запуску.
